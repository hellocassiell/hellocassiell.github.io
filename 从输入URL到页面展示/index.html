从输入URL到页面展示

Posted on 2018-08-20 • In others

当我们输入 URL 并按回车后，浏览器会对 URL 进行检查，首先判断URL格式，比如这些ftp http ed2k，我们这里假设这个URL是https://www.hellocassiell.cn，那么浏览器会对这个 URL 进行安全检查，然后浏览器内核会先查看缓存，再决定是发起请求，下载页面，还是从缓存获取页面，如果是第一次访问，这时候还没有缓存，所以会去向服务器发起一个网络请求。

然后我们看下这个下载页面的过程。
首先我们要确定服务器的网络地址，才能把请求发给它，所以要把这个URL转换成对应的IP地址，浏览器会依次从浏览器缓存，系统缓存，路由器缓存，本地DNS服务器进行一个递归查询，如果本地DNS服务器也没有对应的IP地址的记录，这时候它会把请求发至13台根域名服务器，根域名服务器收到请求后会判断这个域名是谁来授权管理，然后返回一个负责该顶级域名服务器的一个IP。本地DNS服务器收到IP信息后，再去联系顶级域名服务器。顶级域名服务器收到请求后，如果自己无法解析，再返回下一级域名服务器的IP，进行这样一个迭代查询之后，一直到子域名服务器，这个时候有两种情况，第一种是子域名服务器返回了这个URL对应的IP地址，第二种情况是经过了CDN调度，子域名服务器会返回一个CNAME记录，这个记录是CDN服务厂商的一个域名，本地DNS服务器得到这个域名后，会向智能调度DNS查询域名的IP地址，然后智能调度DNS根据一定的算法和策略，返回一个最适合的CDN节点的IP地址，比如说返回一个距离用户最近的CDN节点，因为CDN服务器缓存了网站的源服务器的数据，而且会像源服务器一样，响应用户的请求. 而且它离用户更近，所以能获得更快的响应速度。
但是，知道了IP地址还无法进行通信，因为IP地址并不和某台设备绑定，它只是用来区分不同的计算机是否属于同一个子网络。所以在底层通信时需要使用一个固定的地址，网卡地址，MAC地址，当一台电脑加入网络时，需要通过 ARP 协议告诉其它网络设备它的IP及对应的MAC地址是什么，这样其它设备就能通过 IP 地址来查找对应的设备了。
得到IP后，会建立TCP连接，如果是https开头的URL，还会建立SSL连接，进行一个请求证书，生成对话密钥的过程。
然后开始进行HTTP请求，浏览器会先生成一个HTTP数据包，包含这次请求的信息，主要是请求方法、请求的说明和请求附带的数据，这个HTTP数据包会经过传输层被封装成一个TCP数据包，也就是在这个包的头部加入双方的端口信息，然后经过网络层被封装成一个IP数据包，在这个包的头部加入双方的IP地址信息，然后封装成以太网数据包，头部设置双方的MAC地址，最后变成二进制数据，经过多个网关的转发，传送到服务器。
然后服务器作出响应，返回一个HTTP报文给浏览器，这个HTTP报文包含了状态行，消息报头，消息正文(就是HTML)。
拿到HTML后就是渲染页面的过程。

然后我们再看第二种情况，如果有缓存，浏览器会先判断缓存是否过期，通过Expires和Cache-Control字段来判断是否命中强制缓存，Expires是服务器返回该请求结果缓存的到期时间，Cache-Control的max-age会返回一个相对时间，因为无法确定客户端的时间是否与服务端的时间同步，所以同时存在时，Cache-Control的优先级更高。
如果强制缓存失效，浏览器携带缓存标识向服务器发起请求，由服务器根据缓存标识决定是否使用缓存。
如果上一次 响应设置了ETag值会在这次请求的时候作为If-None-Match的值交给服务器校验：如果一致，继续校验 Last-Modified，发送If-modified-since,如果最后修改时间较新返回200，较旧返回304；不一致返回200。如果没有设置ETag则直接验证Last-Modified，再决定是否返回304。

  • 构建DOM树
    第一遍解析HTML，找出所有HTML 标记转换成文档对象模型 (DOM)；
    遇到外链资源 单独开启下载线程去下载资源：
    1 CSS样式资源 下载 不会阻塞解析HTML；
    2 JS脚本资源 需等待下载完成并执行后才会继续解析HTML 会阻塞；
    2.1 defer与async 脚本变成异步，会等到DOM树构建完后再执行
    3 img图片类资源 异步下载，不会阻塞解析，下载完毕后直接用图片替换原有src的地方；

  • 解析HTML,解析CSS，生成CSSOM树
    第二遍解析HTML，找出CSS 标记转换成 CSS 对象模型 (CSSOM)；

  • 合并DOM树和CSSOM树，生成render树
    从 DOM 树的根节点开始遍历每个可见节点，忽略脚本标记、元标记、display: none的节点；
    对于每个可见节点，为其找到适配的 CSSOM 规则并应用它们；

  • 布局
    从渲染树的根节点开始进行遍历，计算每个节点在网页的确切位置和大小；

  • 绘制
    布局完成后，浏览器会立即发出“Paint Setup”和“Paint”事件，将渲染树转换成屏幕上的像素；

  • 浏览器会将各层的信息发送给GPU，GPU会将各层合成（composite），显示在屏幕上。
